# Домашник — дизайн-манифест

Источник правды по визуальному стилю. Все визуальные правки сверяются с этим документом. Правило репо про токены остаётся железным: НИКАКИХ произвольных цветов/размеров в классах — вся палитра живёт токенами в `src/styles/app.css`, компоненты используют токены и варианты.

## 1. Характер

**«Спокойная уверенность + живая энергия прогресса».** Приложение готовит к стрессовому событию — интерфейс снимает тревогу: много воздуха, крупная дружелюбная типографика, мягкая глубина. Но прогресс — живой и осязаемый: кольца готовности рисуются, серия горит, верный ответ отзывается волной. Не «геймификация-казино», а честная радость движения. Одно дело на экране; всё второстепенное — тише на два тона.

## 2. Палитра (токены)

- **Бренд**: индиго `#5b57e0` (primary, как был) + фирменный **градиент** индиго → фиолет (`#5b57e0 → #8b5cf6`) — токены `--gradient-brand-from/to`; применяется ТОЛЬКО к героям: главный CTA, кольцо готовности, hero-акценты, активный шаг. Не размазывать по мелочи.
- **Светлая тема**: фон слегка тёплый серо-белый; карточки чисто белые; бордеры почти невидимые (граница читается тенью, не линией).
- **Тёмная тема**: НЕ чёрный — глубокий сине-фиолетовый (`#14131f`-семейство), карточки — приподнятый слой (+4-6% светлоты), primary в тёмной чуть светлее/сочнее для контраста.
- **Тепло прогресса**: янтарь/оранж для огня серии (`--flame`), success — тёплый зелёный, warning — янтарный, destructive — приглушённый красный (не кричит).
- Токены завести парами light/dark в `@theme`/`.dark` как уже принято.

## 3. Типографика

- Заголовки: плотные, `tracking-tight`, вес 700–800; h1 крупный (36–44 на десктопе).
- **Цифры-герои** (готовность %, серия, счёт сессии): очень крупные, `tabular-nums`, вес 800 — это фирменные «маяки» экранов.
- Текст: как сейчас (вариант-система Text), вторичный — muted, не уменьшать читаемость.

## 4. Поверхности и глубина

- Радиусы крупные: карточки 16–20px, модалки 24px, пилюли full.
- Тень двойная мягкая (ambient + key): `--shadow-card` (примерно `0 1px 2px rgb(16 14 40 / .04), 0 8px 24px -12px rgb(16 14 40 / .12)`), hover-состояние поднимает (`--shadow-card-hover`). В тёмной теме тени слабее + слой светлоты.
- Иерархия: страница = 1 фон + карточки 1-го уровня + (редко) вложенные плашки `bg-muted/50`. Не громоздить рамку-в-рамке-в-рамке.

## 5. Движение (motion identity)

- **Интерактивные карточки/кнопки**: hover `translateY(-1..2px)` + рост тени; active `scale(.98)`; transition 150–200ms ease-out. Утилиты-классы в app.css (`.lift`, `.press`).
- **Появление списков**: лёгкий stagger fade-up (класс `.rise` + inline animation-delay по индексу; максимум ~8 элементов, дальше без задержки).
- **Кольцо готовности**: при маунте штрих рисуется от 0 до значения (~700ms cubic-bezier); градиентный stroke.
- **Огонь серии**: лёгкое «колыхание» эмодзи/иконки (rotate ±4°, 2.5s) — едва заметно.
- **Верный ответ**: существующий good-pulse остаётся; **«День засчитан»** — одноразовый CSS-конфетти-бёрст (частицы-прямоугольники в брендовых цветах, ~12 штук, 900ms) без библиотек.
- **Переходы карточек в плеере**: текущая уходит fade+slide-left, новая входит slide-right (~250ms).
- ВСЁ уважает `prefers-reduced-motion` (глушить в общем блоке).

## 6. Фирменные элементы

- **ReadinessRing** — главный герой бренда: градиентный штрих, мягкий трек, анимация отрисовки, крупная цифра внутри (tabular, 800). Тёплая (янтарная) при <60%.
- **Прогресс сессии** — сегментированный бар (по карточке на сегмент; отвеченные заливаются, текущая пульсирует).
- **Статусы** — пилюли с цветной точкой (не заливать пилюлю целиком, точка+текст — тише и чище).
- **Пустые состояния** — инлайн-SVG-иллюстрации в дуотоне (индиго-контур + светлая заливка): стопка карточек, календарь с флажком, спокойная луна для bedtime. Небольшие (120–160px), + тёплый текст и один CTA.
- **Иконки** lucide `strokeWidth 1.8`, размер 20 в тексте, 24 в героях.

## 7. Правила по экранам (акценты)

- **Лендинг**: hero с градиентным словом-акцентом, крупные живые демо; секции через 96–128px воздуха; sticky-хедер с blur.
- **Сегодня**: серия+кольца — эмоциональный верх; блоки экзаменов — карточки с lift; план — сегментированный прогресс; CTA «Начать» — градиентный.
- **Плеер**: карточка-«сцена» по центру (max-w-2xl), максимум воздуха, снизу — панель действий; формат-бейдж тихий; уверенность — красивый крупный слайдер с 4 засечками; фидбек: верно — мягкая зелёная волна, неверно — нейтрально-спокойный (никакого «красного стыда» на весь экран).
- **Хаб экзамена**: шапка с большим кольцом + датой; вкладки — сегмент-контрол пилюлей.
- **Статистика**: цифры-герои + чистые бары; калибровка — две колонки «думал/оказалось».
- **Teach**: сцена аватара по центру, диалог — мягкие пузыри; голосовые анимации уже есть — вписать в стиль.
- **Тарифы**: карточка-герой TERM с градиентной рамкой (border-image/обёртка) и бейджем.

## 8. Чего НЕ делать

Глассморфизм-передоз, неоновые свечения, тяжёлые фото/стоковые иллюстрации, 3D-библиотеки, анимации дольше 400ms на действие, цветные заливки больших областей (кроме hero лендинга), новые npm-зависимости ради дизайна, произвольные hex в className.

## 9. Самопроверка скриншотами (обязательна для дизайн-правок)

Headless-скриншоты через playwright из соседнего репо: `cd ~/kvartirnik-app && pnpm exec node <скрипт>` (chromium там установлен) → открыть `http://localhost:3100/...` (dev-сервер Домашника), viewport 390×844 (мобайл) и 1440×900 (десктоп), light и dark (класс dark на html через localStorage theme=dark до загрузки: `page.addInitScript`), полностраничный скриншот в /tmp, ПОСМОТРЕТЬ файл (Read) и поправить найденное. Для приватных страниц — залогиниться fetch-ом (Origin обязан совпадать с BETTER_AUTH_URL).

## 10. Реализация (API дизайн-системы — ЭТАП 1 готов)

Фундамент реализован; ниже — точные имена. Перекраска экранов использует ТОЛЬКО это.

### Токены (`src/styles/app.css`, пары light/dark в `:root`/`.dark`)

- Палитра как раньше (`--background/--card/--primary/--muted/--accent/--success/--warning/--destructive` + `*-foreground`) — значения обновлены: light-фон тёплый, dark — глубокий сине-фиолетовый; ничего в коде менять не надо.
- **Новые цвета**: `--flame` / `--flame-foreground` (огонь серии; классы `text-flame`, `bg-flame`, …), `--gradient-brand-from` / `--gradient-brand-to` (фирменный градиент), `text-brand-foreground` (белый текст на градиенте).
- **Тени**: `--shadow-card`, `--shadow-card-hover`; утилиты `shadow-card`, `shadow-card-hover` (работают с вариантами: `hover:shadow-card-hover`).
- **Радиусы**: `--radius` = 12px → `rounded-lg` кнопки/инпуты; `rounded-2xl` = 20px карточки; `rounded-3xl` = 24px модалки; пилюли `rounded-full`.
- **Кривые**: `--ease-out` (появления/hover), `--ease-in-out` (циклы) — в кастомных анимациях использовать их.
- **Типографика**: `--stat-value-font-size/--stat-value-line-height` — цифры-герои (используются в `Stat`; для ручных героев: `text-(length:--stat-value-font-size) font-extrabold tracking-tight tabular-nums`).

### Утилиты-классы движения (`app.css`, все гасятся в prefers-reduced-motion)

- `.lift` — hover: подъём на 2px + `--shadow-card-hover` (вешать на элементы с `shadow-card`).
- `.press` — active: `scale(0.98)`; уже в базе `Button`.
- `.rise` — fade-up появление; stagger — inline `style={{ animationDelay: … }}` по индексу (максимум ~8 элементов).
- `.flame-sway` — колыхание огня (rotate ±4°, 2.5s) — на иконку/эмодзи серии.
- `.segment-pulse` — пульс текущего сегмента (внутри `SegmentedProgress`).
- `.ring-draw` — отрисовка кольца (внутри `ReadinessRing`).
- `.confetti-burst` — контейнер `absolute inset-0` у родителя с `position: relative`, внутри РОВНО 12 пустых `<span/>`; одноразовый бёрст 900ms — для «День засчитан».
- `.bg-brand-gradient` / `.text-brand-gradient` — градиент фоном / градиентное слово-акцент.
- Существующие `page-enter`, `good-pulse`, `landing-*`, `teach-*` — без изменений, кривые приведены к токенам.

### Компоненты (`~/components`)

- **`Button`**: новый `variant="brand"` — градиентный герой-CTA (белый текст, lift/press, тень); максимум один на экран, обычно с `size="pill"`. Остальные варианты перекрашены под новые радиусы/высоты/фокус-кольца — API прежний.
- **`SimpleCard`**: тень `shadow-card` по умолчанию; новый проп `interactive` (lift+press+cursor-pointer) для кликабельных карточек.
- **`ReadinessRing`**: градиентный штрих + анимация отрисовки от 0 + tabular-цифра; при <60% — тёплый (янтарный) градиент. API прежний (`value` 0..1, `size` sm/md/lg).
- **`SegmentedProgress`** (новый): `<SegmentedProgress total={n} value={answered} />` — сегмент на карточку, отвеченные залиты, текущий пульсирует; при total>40 деградирует в сплошную полосу. `ProgressBar` остаётся для долей (готовность темы, лимиты).
- **`Stat`**: цифра-герой (32px/800/tabular) — API прежний.
- **`Badge`**: новый `variant="dot"` + проп `dot` (`primary|success|warning|destructive|flame|muted`) — статус «цветная точка + тихий текст».
- **`EmptyState`** (новый): `<EmptyState illustration="cards|calendar|moon|map" title text>{CTA}</EmptyState>` — дуотон-SVG 140px (индиго-контур + светлая заливка, темы сами), тёплый текст, слот под один CTA.
- **`PaywallCard`**: освежён (мягкая карточка, accent-плитка иконки, градиентная кнопка) — API прежний.
- **`ResponsiveModal`**: радиус 24px, тень `shadow-card-hover`.
